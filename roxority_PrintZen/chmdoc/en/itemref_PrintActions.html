Settings for Print Views

The following settings can be customized for each Print View by administrative users on the Print Views page in your PrintZen Studio, accessible via Site Settings / PrintZen Studio / Print Views:

"Menu Item and Integration" tab:

Name for this Print View

(Text box)

This name will typically not be visible on content pages to end users, but helps you stay organized.

This Print View is...

(Descriptor for a set of SharePoint Web Sites)

Optionally select in which web sites this Print View will be available. By default, it will be available in all web sites and this usually does not need to be changed during your first explorations within this software. Only perform changes if you are familiar with the terminology presented in the above property editor. Should your changes not result in the expected outcome, you can always easily set this back to "all web sites, no exceptions" and then resume experimenting incrementally with your exceptions.

To define the set of Web Sites, first select either Available in all Web Sites or Available in no Web Sites, then click the except... link to describe which web sites to exclude from that selection. You can exclude:

  • the root Web Site (of the respectively-current Site Collection)
  • Web Sites based on the selected site definitions
  • or Web Sites with the specified titles, URL path fragments or IDs.

Show PrintZen menu item in the following Lists

(Descriptor for a set of SharePoint Lists and Views)

Select in which Lists this Print View will be offered as a menu item to end users (in the SharePoint 2007 "Actions" toolbar menu and in the SharePoint 2010 "View Format" List ribbon group; both only if the "PrintZen Menu Integration" Site Collection feature is activated too and for List Views, only with their Toolbar Type property set to Full or Show, not Summary or None).

To define the set of Lists, first select either Show in all Lists or Show in no Lists, then click the except... link to describe which Lists to exclude from that selection. You can exclude:

  • Lists of the selected base types
  • Lists of the selected template types
  • Lists of the selected base templates
  • or Lists with the specified names or IDs.

To define the set of Views, first select either Show in all Views or Show in no Views, then click the except... link to describe which Views to exclude from that selection. You can exclude:

  • the default View
  • hidden Views
  • personal Views
  • or Views with the specified names or IDs.

Availability on View Item Properties (DispForm.aspx) form pages vs. Views

(Drop-down menu to pick from a set of choices)

By default, the Print View will be offered in the ribbon and tool bar for the Lists selected above, in both the Views selected above and every of its View Item / View Properties (DispForm.aspx) List form pages (if the PrintZen Menu Integration Site Collection feature is activated). This option lets you restrict a Print View to be DispForm-only or View-only (with respect to the ribbon and/or tool bar), if required.

  • Available for both, List item display forms and List Views (as selected above)
  • Available only for List Views (as selected above), not in List item display forms
  • Available only for List item display forms (as selected above), not in List Views

PrintZen menu item caption

(Text box)

Specify the caption to be shown in menus for this Print View. (If you do not specify a caption, the above name for this Print View will be used, or if blank, its internal ID.)

Default value for new Print Views:

$Resources:Tool_PrintActions_DefTitle

PrintZen menu item description

(Text box)

Specify the description to be shown in menus for this Print View.

"Supported Page Modes" tab:

Choose which type of page renders this Print View for this type of print target.

The Original Page mode shows the current source page but hides non-content artifacts (ribbon, navigation, quick launch etc.) and all content artifacts except those targeted by the end user's Print command. This page mode requires the PrintZen Menu Integration Site Collection feature to be activated.

The PrintZen Page mode is a PrintZen-provided custom blank page (showing only the end user-selected print target). If you're planning to use a Custom Design, this is only supported by the PrintZen Page mode.

  • PrintZen page
    — required if you're using a Custom Design for this Print View
  • Original page
    — opens the same page but hides everything except the print target
  • None
    — do not support this target type for this Print View

"Print View and Contents" tab:

This Print View opens

(Drop-down menu to pick from a set of choices)

Select how you would like this Print View to be invoked in end user browsers.

  • in a new pop-up window
    — recommended (even for SharePoint 2010)
  • in a modal pop-up overlay IFRAME
    — (SharePoint 2010 only, not available in WSS 3.0 / MOSS 2007)
  • in the current browser tab

In the Print View, show a Print Now button and brief instructions

(Check box option)

This will not be included in the printed output, but may prove useful to your end users.

If you selected above that this Print View opens in a modal pop-up overlay IFRAME (SharePoint 2010 only), then this option cannot be turned off and an info bar with a Print button will always be displayed in the Print View (but never in the final print-out).

Show the browser's printer selection dialog immediately

(Check box option)

If ticked, this option invokes the Print dialog automatically as soon as this Print View is opened.

[List Views with PrintZen page mode] Show current View

(Check box option)

If ticked, this Print View only ever includes items included in the currently displayed List View (rather than the default View), applying the View-defined default sort orders and filters (but not currently user-applied live sort order, filters or paging); otherwise, all List items in the default View are displayed, pre-filtered and in the sort order prescribed by that View.

[List Views with PrintZen page mode] Apply current sort order

(Check box option)

If ticked, uses the current sort order applied via the Column headers by the end user (if any), rather than the one pre-defined in the View.

[List Views with PrintZen page mode] Display folder contents

(Drop-down menu to pick from a set of choices)

Choose how you want the current folder selected by the end user (if any) to be taken into account.

  • Show only the items in the current folder, but no sub-folders
  • Show only the items and folders in the current folder
  • Show no folders, but all items from all folders and sub-folders
  • Show all items, folders and sub-folders in this List

[List Views with PrintZen page mode] Apply current paging

(Check box option)

If ticked, uses the current paging position of the end user, if any.

[List Views with PrintZen page mode] Apply current filters (FilterZen only)

(Check box option)

If ticked, for List Views this Print Views only ever shows List items included in the filtering currently applied to the selected source List View. (Only supports filters supplied by a currently connected FilterZen Filter Web Part.)

"Custom Design" tab:

Web Part page for Custom Design of Print View

(Text box)

ONLY supported for Print Views using the PrintZen page mode (see Supported Page Modes tab)!

By default, the Print View contains nothing but the selected print target(s). To include additional contents, you can create a custom Web Part Page (only the Full Page, Vertical Layout Template is supported) and specify its absolute or relative URL here (for example, /Shared Documents/OurPrintView.aspx). In this case, the print view will attempt to load and include all Web Parts on that page. That page will also need one additional placeholder Web Part of any kind (for example, a Content Editor Web Part, empty or not) with the title set precisely to PRINTZEN_TARGET, to indicate where the print target(s) should be located. This placeholder Web Part will not be included in the print-out, in its place, the print targets will be added instead.

Note: not all Web Parts may display as expected, since the Web Parts will be "temporarily cloned" in a different page context that is not technically a full-fledged Web Part page — so experiment for yourself. Web Part Connections, for example, are not currently supported.

Include Web Part page title bar

(Check box option)

By default, PrintZen does not include the Title Bar Web Part that is almost always included in all custom Web Part pages, but very often not customized. Tick this option if you'd like to include the Title Bar of the above Web Part page in this Print View.

Show placeholder names with current values

(Check box option)

Shows in this Print View all placeholders that you can use for Web Part properties throughout the above custom Web Part page. (Not advisable for end-user printing, but temporarily highly useful while designing your custom Web Part page.)

Which placeholders are available and their values both greatly depend on the current context of the Web Part being printed. For your general reference, the following placeholders may usually be available, but we advise you to inspect their typical values and availability in your context interactively yourself by ticking this check-box option during the design phase.

  • {$PrintZen_WebPart_Title$} — the title of the Web Part being printed (all modes)
  • {$PrintZen_List_Title$} — the title of the List being printed in List View mode (or the List containing the item properties being printed, in DispForm.aspx mode)
  • {$PrintZen_Item_Title$} — the title of the List item whose properties being printed (if in DispForm.aspx mode)
  • {$PrintZen_View_Title$} — the title of the View being printed (if in List View mode)
  • {$PrintZen_Context_Title$} — the title of the List Item, List and View, or Web Part, depending on mode
  • {$PrintZen_PrintAction_xyz$} — instead of xyz, various placeholders containing most settings of this Print View
  • {$PrintZen_QueryString_a$} — the ID of this Print View
  • {$PrintZen_QueryString_l$} — the ID of the Web Part being printed (in PeopleZen mode) or the ID of the List (in List View and DispForm.aspx modes)
  • {$PrintZen_QueryString_View$} — the View ID (in List View mode), item ID (in DispForm.aspx mode) or the source Web Part page URL (in PeopleZen mode)
  • {$PrintZen_TitleBar_Caption$} — the caption set for the Title Bar Web Part of the Web Part page specified above.
  • {$PrintZen_TitleBar_Description$} — the description set for the Title Bar Web Part of the Web Part page specified above.
  • {$PrintZen_TitleBar_Title$} — the title set for the Title Bar Web Part of the Web Part page specified above.
  • {$PrintZen_TitleBar_Image$} — the image set for the Title Bar Web Part of the Web Part page specified above.

Add a "ScriptManager" control to the page (NOT recommended)

(Check box option)

Under very rare and unlikely circumstances, using certain (non-ROXORITY) 3rd-party Web Parts on your custom design page will result in an error message that a ScriptManager control is required ("The control requires a ScriptManager on the page. The ScriptManager must appear before any controls that need it.").

ONLY if you do receive the above error message should you tick this option: because if you tick it when it is not required, this will give your end users another kind of script warning and you'll want to avoid this.
