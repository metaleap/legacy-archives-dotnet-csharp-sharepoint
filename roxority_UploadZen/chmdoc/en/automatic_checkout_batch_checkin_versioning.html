Automatic Check-Out & Batch Check-In

UploadZen supports check-in and check-out for Document Libraries that have the Require documents to be checked out before they can be edited setting set to Yes (on the Versioning Settings page under Document Library Settings).

UploadZen Web Uploader

You can select for each file whether and how it should be checked in after it was successfully uploaded:

  • No — uploaded files stay checked out to you
  • As minor new version — uploaded files are checked in. Overwritten, previously existing files get a minor version number increment (ie. from 1.0 to 1.1 etc.) if the Document Version History for this Document Library is set to Create major and minor (draft) versions (on the Versioning Settings page under Document Library Settings).
  • As major new version — uploaded files are checked in. Overwritten, previously existing files get a manor version number increment (ie. from 1.0 to 2.0 etc.) unless the Document Version History for this Document Library is set to No versioning (on the Versioning Settings page under Document Library Settings).
  • Overwrite current version — uploaded files are checked in but if the file name already existed prior to the upload, the previous version number remains unchanged.

To check in all files the same way, select the preferred setting for one file, then click the icon to Set all files to this selection.

UploadZen Windows Uploader

In the legacy Windows Uploader, prior to your upload simply drop-down the menu next to the Upload Files button:

  • Auto Check-Out — tick this menu option to check out existing files to you prior to overwriting them. This is necessary or existing files cannot be overwritten.
  • Auto Check-In — select whether uploaded files should be checked in after they have been uploaded:
    • None — uploaded files stay checked out to you
    • Minor versions — uploaded files are checked in. Overwritten, previously existing files get a minor version number increment (ie. from 1.0 to 1.1 etc.) if the Document Version History for this Document Library is set to Create major and minor (draft) versions (on the Versioning Settings page under Document Library Settings).
    • Major versions — uploaded files are checked in. Overwritten, previously existing files get a manor version number increment (ie. from 1.0 to 2.0 etc.) unless the Document Version History for this Document Library is set to No versioning (on the Versioning Settings page under Document Library Settings).
    • Major versions — uploaded files are checked in but if the file name already existed prior to the upload, the previous version number remains unchanged.
