  • Nie
  • Nur wenn Upload-Aktionen existieren (bei aktiviertem Websitesammlungsfeature)
  • Immer (bei aktiviertem Websitesammlungsfeature)

Alternativer Pfad für temporäre Ablage von Dateistückelungen

Sie müssen zwingend einen alternativen Pfad festlegen, wenn Sie die Dateistückelungs-Funktion (Chunking, aktivierbar pro Upload-Aktion) in einer Multiserver-Farmumgebung einsetzen wollen. In einer Einzelserver-Farm oder bei Verzicht auf die Stückelungs-/Chunkingfunktion kann diese Einstellung ignoriert werden. Andernfalls lesen Sie zunächst alle Details im Abschnitt Preparing a Web Application for the Chunking feature des UploadZen-Hilfethemas Deployment-Checkliste.

Wenn diese Einstellung nicht gesetzt ist, wird standardmäßig der physikalische Verzeichnispfad verwendet, der dem virtuellen Verzeichnispfad _layouts/roxority_UploadZen/fc entspricht. Falls diese Einstellung gesetzt werden soll, müssen Sie diese auf einen von allen Web-Frontendservern zentral zugänglichen Speicherort setzen, z.B. ein Netzwerklaufwerk, gemeinsamen Order oder ähnliches.

Fehlermeldungen enthalten immer sämtliche Debug- und Diagnosedetails

Standardmäßig wird für bestimmte von UploadZen abgefangene und an Endanwender weitergegebene Fehler nur die Fehlermeldung angezeigt. Zu Debug- und Diagnosezwecken kann es indes mitunter temporär vorteilhaft sein, alle bekannten Fehlerdetails anzuzeigen (wie z.B. innere Ausnahmen oder die Stapelverfolgung), im laufenden Dauerbetrieb jedoch würden solche detaillierten Meldungen Ihre Anwender verwirren oder verschrecken. Kreuzen Sie diese Option an, um immer alle bekannten Fehlerdetails in Ihren Endbenutzerschnittstellen anzuzeigen.

Keine Standardinstanzen für UploadZen-Konfiguratoren auf Websitesammlungsebene (NICHT empfohlen!)

Konfiguratoren sind Listen von Elementen, die das Verhalten dieser Software definieren bzw. anpassen (wie Export-Aktionen in ExportZen, Datenquellen und Datenfeldvorlagen in PeopleZen, Druckansichten und Druck-Launcher in PrintZen, Upload-Aktionen in UploadZen). Standardmäßig besitzt jede Websitesammlung für jeden Konfigurator ihre eigene, nicht entfernbare Standardinstanz , die angepaßt werden kann oder zu der weitere hinzugefügt werden können. In sehr großen Serverfarmen mit 1000en von Websitesammlungen können diese Standardinstanzen eher hinderlich sein und der ausschließliche Einsatz von Instanzen auf Farmebene präferiert werden. Kreuzen Sie diese Option an, um die Standardinstanz-pro-Websitesammlung zu deaktivieren.

Standardmäßig keine Einbindung der jQuery-Bibliothek v1.8.3 (NICHT empfohlen)

Einige UploadZen-Benutzeroberflächen erfordern die jQuery-Bibliothek und binden diese in die entsprechenden Seiten ein. Mehrfacheinbindung von jQuery kann manchmal zu Problemen führen. Kreuzen Sie auf der Seite Konfiguration im UploadZen-Studio diese Option an nur, wenn Ihre Masterseiten, Webpartseiten oder Drittanbietererweiterungen eigenständig, global (nicht nur in Einzelfällen) und unabhängig von UploadZen ebenfalls jQuery einbinden, um UploadZen davon abzuhalten, jQuery einzubinden. Ist diese Option aktiviert, müssen Sie selbständig sicherstellen, dass jQuery (in einer Version neuer als 1.4 ABER älter als 1.9) in Seiten geladen wird, die UploadZen-Benutzeroberflächen enthalten, andernfalls wird UploadZen unter Umständen nicht mehr funktionieren. Dies ist eine globale Voreinstellung, die lokal ggf. noch einmal individuell überschrieben werden kann.
