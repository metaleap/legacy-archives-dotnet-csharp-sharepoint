Die folgenden Einstellungen finden administrative Benutzer auf der Seite Konfiguration im PeopleZen-Studio, welches Sie über Websiteeinstellungen / PeopleZen Studio / Konfigurationseinstellungen erreichen.

Jede Einstellung hat einen globalen, über die gesamte Serverfarm hinweg geltenden Wert, der jeweils optional auf Websitesammlungsebene überschrieben werden kann. Die folgenden Konfigurationseinstellungen stehen zur Verfügung:

Sprache

PeopleZen wurde für verschiedene Sprachen lokalisiert. Zusätzlich zu Englisch und Deutsch werden hier alle Sprachen zur Auswahl angeboten, für die Sie eigene Teilübersetzungen (unter Websiteeinstellungen / PeopleZen Studio / DIY-Benutzeroberflächen-Lokalisierung) angefertigt haben. Diese Einstellung betrifft alle PeopleZen-Benutzeroberflächen außer Beschriftungen und Beschreibungen von Features, Listenvorlagen, Spaltentypen, Inhaltstypen oder Webpartvorlagen (diese werden von SharePoint immer direkt gemäß der aktiven Website-Spracheinstellung geladen).

Wählen Sie aus folgenden Optionen auf der Seite Konfiguration im PeopleZen-Studio:

  • (Wie Site, oder Englisch)
  • English
  • Deutsch

Fensterziel für Name/Titel-Links von Datensätzen

Vom PeopleZen-Webpart pro Datensatz generierte "Name/Titel"-Links werden in einem HTML-Zielfenster Ihrer Wahl geöffnet. Verwenden Sie _self, um diese Links im selben Browserfenster zu öffnen, _blank, um sie in einem neuen Browserfenster zu öffnen, _modal, um sie in einem modalen Popup-Dialogfenster zu öffnen, _popup für ein "modales" seitenintegriertes Overlay-"Popup", oder geben Sie den Namen eines IFRAME- oder FRAME-Elements in Ihrer Webpartseite an.

Falls Sie diese Einstellung angepaßt haben, können Sie über den Link Auf Standardwert zurücksetzen auf der Seite Konfiguration im PeopleZen-Studio folgenden Standardwert wiederherstellen:

_popup

HTML-Vorlage für E-Mail-Adressen

Die vom PeopleZen-Webpart zu generierende HTML-Ausgabe für alle Datensatzfeldwerte, die E-Mail-Adressen repräsentieren. Gültige Platzhalter: {0} für die E-Mail-Adresse, {1} für { und {2} für }.

Falls Sie diese Einstellung angepaßt haben, können Sie über den Link Auf Standardwert zurücksetzen auf der Seite Konfiguration im PeopleZen-Studio folgenden Standardwert wiederherstellen:

<a href="mailto:{0}">{0}</a>

HTML-Vorlage für Datensatzfeld ohne Wert

Falls Sie diese Einstellung angepaßt haben, können Sie über den Link Auf Standardwert zurücksetzen auf der Seite Konfiguration im PeopleZen-Studio folgenden Standardwert wiederherstellen:

Alle Datensatzfelder ohne Wert ausschließen

Standardmäßig generiert das PeopleZen-Webpart (in der Kachel-Ansicht) ein HTML-Blockelement für jedes in den Webpart-Einstellungen angegebene Datenfeld, selbst wenn für das Feld im jeweiligen Datensatz (noch) kein Wert festgelegt wurde. Kreuzen Sie auf der Seite Konfiguration im PeopleZen-Studio diese Option an, um für leere Datensatzfelder überhaupt keine HTML-Ausgabe (in der Kachel-Ansicht) zu generieren.

Schwellenwert für Reduzierung der Schnellfilter-Reiter

Schnellfilter-Reiter werden so generiert, daß jeder Reiter einem existierenden Feldwert entspricht. Dies ist vorteilhaft bei Mehrfachfeldern (wie z.B. Fähigkeiten, Verantwortlichkeiten etc. bei Farm-Benutzerprofilen), aber nicht zielführend bei potentiell großen Wertemengen (Vorname, Nachname...) — daher werden bei Erreichen einer bestimmten Gesamtanzahl von Feldwerten die Reiter auf alphabetische oder numerische Indizes (A, B, C...) reduziert.

Geben Sie diese Höchstanzahl hier an. Geben Sie keine gültige Ganzzahl zwischen 2 und 50 ein, wird der Standardwert 16 verwendet.

Falls Sie diese Einstellung angepaßt haben, können Sie über den Link Auf Standardwert zurücksetzen auf der Seite Konfiguration im PeopleZen-Studio folgenden Standardwert wiederherstellen:

16

Den Schnellfilter-Reiter [Leer] immer ausblenden

Standardmäßig wird immer ein Schnellfilter-Reiter zur Anzeige der Datensätze mit fehlendem / leerem Feldwert hinzugefügt (aber nur, falls solche existieren). Kreuzen Sie auf der Seite Konfiguration im PeopleZen-Studio diese Option an, um diesen Schnellfilter-Reiter nie anzuzeigen.

Den Schnellfilter-Reiter [Alle] immer einblenden

Bei der Verwendung von Schnellfilter-Reitern wird standardmäßig der erste Reiter aktiviert, und Anwender haben keine Möglichkeit, "alle Datensätze, ungefiltert" auf einmal anzuzeigen. Kreuzen Sie auf der Seite Konfiguration im PeopleZen-Studio diese Option an, um einen weiteren [Alle]-Schnellfilter-Reiter anzuzeigen, der dies ermöglicht und dann auch voraktiviert wird.

Schnellfilter-Reiter immer in umgekehrter Sortierreihenfolge darstellen

Kreuzen Sie diese Option an, um die Schnellfilter-Reiter immer in absteigender (statt aufsteigender) Reihenfolge sortiert anzuzeigen.

Internationale Zeichen in Schnellfilter-Tabs normalisieren

Experimentieren Sie mit dieser Einstellungen, wenn die Ihren Schnellfilter-Tabs zugrundeliegenden Werte nicht-ASCII / internationale Zeichen enthalten und Ihre Schnellfilter-Tabs unerwartet, seltsam oder unbenutzbar erscheinen.

Einbettung verbundener FilterZen-Webparts

Standardmäßig wird der interaktive Filterbereich eines FilterZen-Webparts visuell in das verbundene PeopleZen-Webpart eingebettet (nur bei für letzteres aktivierter Ajax-Livefilterung). Dies verbessert oft die Benutzerfreundlichkeit, aber Sie können dieses Verhalten hier bei Bedarf auch deaktivieren.

Wählen Sie aus folgenden Optionen auf der Seite Konfiguration im PeopleZen-Studio:

  • Ja (ohne oberen Rahmen)
  • Ja
  • Nein

Lokaler Textvergleichsmodus für bestimmte Datenquellen-Verbindungsanbieter

Für nicht datenquellenseitig, sondern SharePoint-seitig durch PeopleZen selbst durchgeführte Filterungen können Sie hier den Textvergleichsalgorithmus auswählen, der zur Filterung anhand von textuellen (statt numerischen oder datumsbasierten) Datenfeldwerten verwendet wird.

Wählen Sie aus folgenden Optionen auf der Seite Konfiguration im PeopleZen-Studio:

  • Regionsneutral
  • Aktuelle Regionseinstellung
  • Ordinal

AJAX-basiertes "träges Laden" von Datensätzen erst nach Anzeige der Seite (normalerweise nicht vorteilhaft)

Falls nicht hierunter deaktiviert, werden bestimmte Endanwender-Interaktionen (vor- oder zurückblättern, Umsortierung, Umgruppierung oder Neufilterung) von PeopleZen auf AJAX-Basis ausgeführt, ohne daß die gesamte Seite für diese Interaktion jedes Mal neu geladen werden muß — beim Erstaufruf der Seite wird die Initialanzeiges des PeopleZen-Webparts jedoch gleich während des Seitenaufbaus geladen und vorbereitet. Solange dies nicht zu signifikanten (!) Performanceeinbußen oder stark verlängerte Seitenladezeiten führt, ist dies auch der bevorzugte und benutzerfreundlichere Ansatz. Kreuzen Sie diese Option an, um die PeopleZen-Initialanzeige nie schon während des Seitenaufbaus vorzubereiten und diese stattdessen erst nach dem vollständigen clientseitigen Laden der Seite über AJAX "träge" nachzuladen.

AJAX-basiertes Verhalten deaktivieren (nicht empfohlen)

Bestimmte Endanwender-Interaktionen (vor- oder zurückblättern, Umsortierung, Umgruppierung oder Neufilterung) werden von PeopleZen auf AJAX-Basis ausgeführt, ohne daß die gesamte Seite für diese Interaktion jedes Mal neu geladen werden muß. Dieses generell benutzerfreundlichere Verhalten funktioniert jedoch in einigen seltenen FBA/ISA-Konstellationen oder anderen wenigen hochangepaßten Enterprise-VPN bzw. Proxy-beeinflußten Umgebungen leider nicht. Kreuzen Sie auf der Seite Konfiguration im PeopleZen-Studio diese Option an, um alle AJAX-basierten Interaktionen umzustellen auf den klassischen, Seite-neuladenden "Postback"-Benutzerinteraktionsmodus.

Keine Standardinstanzen für PeopleZen-Konfiguratoren auf Websitesammlungsebene (NICHT empfohlen!)

Konfiguratoren sind Listen von Elementen, die das Verhalten dieser Software definieren bzw. anpassen (wie Export-Aktionen in ExportZen, Datenquellen und Datenfeldvorlagen in PeopleZen, Druckansichten und Druck-Launcher in PrintZen, Upload-Aktionen in UploadZen). Standardmäßig besitzt jede Websitesammlung für jeden Konfigurator ihre eigene, nicht entfernbare Standardinstanz , die angepaßt werden kann oder zu der weitere hinzugefügt werden können. In sehr großen Serverfarmen mit 1000en von Websitesammlungen können diese Standardinstanzen eher hinderlich sein und der ausschließliche Einsatz von Instanzen auf Farmebene präferiert werden. Kreuzen Sie diese Option an, um die Standardinstanz-pro-Websitesammlung zu deaktivieren.

Standardmäßig keine Einbindung der jQuery-Bibliothek v1.8.3 (NICHT empfohlen)

Einige PeopleZen-Benutzeroberflächen erfordern die jQuery-Bibliothek und binden diese in die entsprechenden Seiten ein. Mehrfacheinbindung von jQuery kann manchmal zu Problemen führen. Kreuzen Sie auf der Seite Konfiguration im PeopleZen-Studio diese Option an nur, wenn Ihre Masterseiten, Webpartseiten oder Drittanbietererweiterungen eigenständig, global (nicht nur in Einzelfällen) und unabhängig von PeopleZen ebenfalls jQuery einbinden, um PeopleZen davon abzuhalten, jQuery einzubinden. Ist diese Option aktiviert, müssen Sie selbständig sicherstellen, dass jQuery (in einer Version neuer als 1.4 ABER älter als 1.9) in Seiten geladen wird, die PeopleZen-Benutzeroberflächen enthalten, andernfalls wird PeopleZen unter Umständen nicht mehr funktionieren. Dies ist eine globale Voreinstellung, die lokal ggf. noch einmal individuell überschrieben werden kann.
