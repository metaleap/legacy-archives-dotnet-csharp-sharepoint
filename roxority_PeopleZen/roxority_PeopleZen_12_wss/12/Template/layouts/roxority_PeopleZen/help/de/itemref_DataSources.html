Register "Caching":

Datenquellen-Caching im Server-RAM

(Dropdown-Menü zur Auswahl einer Option)

Wird Caching aktiviert, erstellt PeopleZen beim ersten Aufruf dieser Datenquelle (nachdem ein Webfrontend-Server zurückgesetzt wurde) immer einen vollen Cache aller Datensätze im Server-RAM.

Achtung: bei aktiviertem Caching werden mitunter "veraltete" Daten angezeigt. Datensatzänderungen werden nicht in Echtzeit, sondern verzögert wiedergegeben (siehe Aktualisierungsrate für Cache hierunter bei aktiviertem Caching). Vollständig neue Datensätze werden ebenfalls nicht bis zum nächsten Server-Reset aufgegriffen, außer Sie kreuzen die Option Cache regelmäßig komplett neu aufbauen (hierunter bei aktiviertem Caching) an.

Falls diese Datenquelle den Verbindungsanbieter Datenbank verwendet und Sie für sie Caching aktiviert haben, müssen Sie die Hinweise der ADO.NET-Datenabfrage-Einstellung (im Datenbank-Register) sowie deren aktuellen Wert untersuchen und sicherstellen, daß Ihre Abfrage immer und zu jeder Zeit eine konsistente Sortierreihenfolge der Ergebnismenge garantiert.

  • Volles Caching
    — Daten werden aus (ggf. etwas veralteter) RAM-Kopie ALLER Backend-Datensätze gelesen
  • Kein Caching (empfohlen)
    — jede Anfrage verbindet sich mit dem Backend-Speicher dieser Datenquelle

Aktualisierungsrate für Cache

(Texteingabefeld)

Nachdem ein Webfrontend-Server zurückgesetzt wurde, erstellt PeopleZen beim ersten Aufruf dieser Datenquelle immer einen vollen Cache aller Datensätze im Server-RAM.

Bei jedem darauf folgenden Aufruf aktualisiert PeopleZen immer nur eine begrenzte Anzahl an Datensätzen im Cache, um Performance mit Korrektheit der angezeigten Informationen auszubalancieren.

Geben Sie hier die Anzahl der pro Aufruf zu aktualisierenden Datensätze an. Eine höhere Zahl (z.B. ca. 40 oder mehr) stellt Datenänderungen eher dar bei reduzierter PeopleZen-Geschwindigkeit, und umgekehrt stellt ein geringerer Wert Datenänderungen erst sehr viel später fest bzw. dar, bei verbesserter Geschwindigkeit.

Geben Sie 0 an, um gecachte Datensätze nie während der Datensatzabfrage durch Endanwender zu aktualisieren. Geben Sie keine gültige Ganzzahl größer gleich 0 ein, so wird der Standardwert 10 verwendet.

Standardwert für neue Datenquellen:

10

Aktuell angezeigte Cache-Datensätze immer bei Abruf aktualisieren

(Ankreuzbares Kontrollfeld)

Aktualisiert immer die aktuell abgerufenen Datensätze (gerade vor deren Anzeige, ABER *nach* Filterung/Gruppierung/Sortierung!), bei jedem einzelnen Datensatzabruf durch Endanwender; dies jedoch mit möglicherweise spürbaren Performanceeinbußen bei höheren Paging-Seitengrößen.

Cache regelmäßig komplett neu aufbauen

(Ankreuzbares Kontrollfeld)

In Produktivsystemen normalerweise weder erforderlich noch empfehlenswert, außer es finden "häufige" (>1x pro Tag) Löschungen und Neuerstellungen von Datensätzen im Backend-Speicher dieser Datenquelle statt.

Nachdem ein Webfrontend-Server zurückgesetzt wurde, erstellt PeopleZen beim ersten Aufruf dieser Datenquelle immer einen vollen Cache aller Datensätze im Server-RAM.

Die Datensätze im Cache werden zwar periodisch aktualisiert wie oben angegeben; komplett neue Datensätze oder Löschungen werden jedoch bis zum nächsten Server-Reset hier nicht mit einbezogen, es sei denn:

Aktivieren Sie diese Option, um bei jedem n-ten Webpart-Aufruf den Cache wie nach einem Server-Reset komplett neu aufzubauen, wodurch jeder n-te Webpart-Aufruf stark verlangsamt wird, aber auch seit dem letzten Cacheaufbau neu hinzugefügte Datensätze mit einbezogen werden (n = Gesamtanzahl der Datensätze geteilt durch oben angegebene Aktualisierungsrate).

Den Cache beim nächsten Klick auf "Alle speichern" leeren

Leert beim nächsten Klick auf "Alle speichern" den Cache dieser Datenquelle (nur auf dem empfangenden WFE-Server), um Änderungen an Backend-Daten oder Datenfeldvorlagen sofort aufzugreifen.

Secondary sort-by field name

(Texteingabefeld)

Primary sorting is defined per Web Part (or other Data Source consumer) and may or may not be set interactively by end users depending on said configuration.
