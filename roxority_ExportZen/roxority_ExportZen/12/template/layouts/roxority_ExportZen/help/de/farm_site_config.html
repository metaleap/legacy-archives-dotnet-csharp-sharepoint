Die folgenden Einstellungen finden administrative Benutzer auf der Seite Konfiguration im ExportZen-Studio, welches Sie über Websiteeinstellungen / ExportZen Studio / Konfigurationseinstellungen erreichen.

Jede Einstellung hat einen globalen, über die gesamte Serverfarm hinweg geltenden Wert, der jeweils optional auf Websitesammlungsebene überschrieben werden kann. Die folgenden Konfigurationseinstellungen stehen zur Verfügung:

Sprache

ExportZen wurde für verschiedene Sprachen lokalisiert. Zusätzlich zu Englisch und Deutsch werden hier alle Sprachen zur Auswahl angeboten, für die Sie eigene Teilübersetzungen (unter Websiteeinstellungen / ExportZen Studio / DIY-Benutzeroberflächen-Lokalisierung) angefertigt haben. Diese Einstellung betrifft alle ExportZen-Benutzeroberflächen außer Beschriftungen und Beschreibungen von Features, Listenvorlagen, Spaltentypen, Inhaltstypen oder Webpartvorlagen (diese werden von SharePoint immer direkt gemäß der aktiven Website-Spracheinstellung geladen).

Wählen Sie aus folgenden Optionen auf der Seite Konfiguration im ExportZen-Studio:

  • (Wie Site, oder Englisch)
  • English
  • Deutsch

SharePoint's vordefinierte "Nach Excel exportieren"-Originalaktion deaktivieren

Standardmäßig koexistieren Ihre ExportZen-Aktionen parallel zum bereits in SharePoint eingebauten 'Nach Excel exportieren'-Originalbefehl. Wollen Sie letztere Funktionalität komplett durch ExportZen ersetzen, können Sie hier anpassen, ob und wann SharePoints eigener Excel-Exportbefehl deaktiviert werden soll (betrifft sowohl den Menübefehl in der Aktionen-Toolbar von SharePoint 2007-Listen als auch die Ribbon-Befehle der 'Verbinden und exportieren'-Gruppe im "Liste"-Ribbon von SharePoint 2010/2013).

Wählen Sie aus folgenden Optionen auf der Seite Konfiguration im ExportZen-Studio:

  • Nie
  • Nur wenn CSV-Exportaktionen existieren (bei aktiviertem Websitesammlungsfeature)
  • Immer (bei aktiviertem Websitesammlungsfeature)

Keine Standardinstanzen für ExportZen-Konfiguratoren auf Websitesammlungsebene (NICHT empfohlen!)

Konfiguratoren sind Listen von Elementen, die das Verhalten dieser Software definieren bzw. anpassen (wie Export-Aktionen in ExportZen, Datenquellen und Datenfeldvorlagen in PeopleZen, Druckansichten und Druck-Launcher in PrintZen, Upload-Aktionen in UploadZen). Standardmäßig besitzt jede Websitesammlung für jeden Konfigurator ihre eigene, nicht entfernbare Standardinstanz , die angepaßt werden kann oder zu der weitere hinzugefügt werden können. In sehr großen Serverfarmen mit 1000en von Websitesammlungen können diese Standardinstanzen eher hinderlich sein und der ausschließliche Einsatz von Instanzen auf Farmebene präferiert werden. Kreuzen Sie diese Option an, um die Standardinstanz-pro-Websitesammlung zu deaktivieren.

Standardmäßig keine Einbindung der jQuery-Bibliothek v1.8.3 (NICHT empfohlen)

Einige ExportZen-Benutzeroberflächen erfordern die jQuery-Bibliothek und binden diese in die entsprechenden Seiten ein. Mehrfacheinbindung von jQuery kann manchmal zu Problemen führen. Kreuzen Sie auf der Seite Konfiguration im ExportZen-Studio diese Option an nur, wenn Ihre Masterseiten, Webpartseiten oder Drittanbietererweiterungen eigenständig, global (nicht nur in Einzelfällen) und unabhängig von ExportZen ebenfalls jQuery einbinden, um ExportZen davon abzuhalten, jQuery einzubinden. Ist diese Option aktiviert, müssen Sie selbständig sicherstellen, dass jQuery (in einer Version neuer als 1.4 ABER älter als 1.9) in Seiten geladen wird, die ExportZen-Benutzeroberflächen enthalten, andernfalls wird ExportZen unter Umständen nicht mehr funktionieren. Dies ist eine globale Voreinstellung, die lokal ggf. noch einmal individuell überschrieben werden kann.
